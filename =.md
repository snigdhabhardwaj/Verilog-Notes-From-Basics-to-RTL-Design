# Thought Experiment #2: One Character, Completely Different Hardware Behavior

Take a look at this code:

always @(posedge clk) begin
    a = b;
    c = a;
end

## Before You Read Further

Question: Assuming the values before the clock edge are:

| Register | Value |
| -------- | ----: |
| `a`      |   `0` |
| `b`      |   `1` |
| `c`      |   `0` |

What values do you expect after the clock edge?

Take a moment to predict the result before continuing.

## What Actually Happens?

With **blocking assignments (`=`)**, statements execute one after another **within the procedural block**.

First,

```verilog
a = b;
```

updates `a` immediately.

Now `a` becomes `1`.

Then,

```verilog
c = a;
```

reads the updated value of `a`.

The final values become:

| Register | Value |
| -------- | ----: |
| `a`      |   `1` |
| `b`      |   `1` |
| `c`      |   `1` |

Unlike the previous example, `c` now receives the **new** value of `a`.

---

## Why Is This Dangerous?

Compare these two pieces of code:

```verilog
// Version A
always @(posedge clk) begin
    a <= b;
    c <= a;
end
```

```verilog
// Version B
always @(posedge clk) begin
    a = b;
    c = a;
end
```

Only one character changes:
<=
becomes
=

Yet the behavior is completely different.

Version A models **two independent flip-flops** sampling simultaneously.

Version B behaves like `a` changes before `c` reads it, which is **not** how two independent flip-flops connected to the same clock behave.

That's why using blocking assignments inside sequential logic can produce simulation results that don't accurately represent the intended hardware.

---

## Build the Hardware Intuition

When you see:

```verilog
always @(posedge clk)
```

ask yourself one question:

> **"Am I describing hardware or am I accidentally writing software?"**

If you're describing registers triggered by a clock, each register should capture its input independently at the clock edge.

Non-blocking assignments naturally model that behavior.

Blocking assignments make the code behave like a sequence of software statements, which can hide bugs and create misleading simulation results.

---

## Mental Model to Remember

> **Use non-blocking assignments (`<=`) to describe what registers capture on a clock edge. Use blocking assignments (`=`) for combinational logic, where statements are meant to execute sequentially within the procedural block.**
